# Guia de Estudo: Markdown - O que é e Como Funciona?

## Quiz (10 Perguntas de Resposta Curta)

 1.⁠ ⁠*O que é Markdown e qual é sua principal função?*  
   Markdown é uma linguagem de marcação que utiliza um conjunto de sinais para formatar textos. Sua principal função é simplificar a formatação de textos, especialmente para a parte visual (front-end) de websites, aplicativos ou documentos, de uma forma mais prática que o HTML.

 2.⁠ ⁠*Markdown substitui o HTML? Explique sua relação.*  
   Não, o Markdown não substitui o HTML. Ele é um complemento, utilizado especificamente para a construção e formatação de textos (títulos, parágrafos, listas, etc.), enquanto o HTML é uma linguagem mais abrangente para a estruturação e conteúdo geral de uma página web.

 3.⁠ ⁠*Cite três tipos de formatação de texto que podem ser feitos com Markdown e o sinal correspondente para cada um.*  
   Alguns exemplos incluem:  
   - Títulos: ⁠ # ⁠  
   - Negrito: ⁠ **texto** ⁠  
   - Itálico: ⁠ *texto* ⁠  
   - Listas Ordenadas: ⁠ 1. item ⁠

 4.⁠ ⁠*Em que tipo de ambiente de desenvolvimento o vídeo menciona que o Markdown é frequentemente utilizado para formatar células de texto?*  
   O vídeo menciona o *Jupyter Notebook* como um ambiente onde o Markdown é amplamente utilizado. Nele, é possível alterar o tipo de célula para "Markdown" e escrever textos formatados.

 5.⁠ ⁠*Qual é a diferença entre um título H1, H2 e H3 no Markdown, em termos de como eles são indicados?*  
   A diferença está no número de sinais de hashtag ⁠ # ⁠ utilizados.  
   - H1: ⁠ # ⁠  
   - H2: ⁠ ## ⁠  
   - H3: ⁠ ### ⁠  
   Quanto mais ⁠ # ⁠, menor a hierarquia e o tamanho do título.

 6.⁠ ⁠*Como o vídeo compara a complexidade de formatar um título em Markdown versus HTML?*  
   O vídeo demonstra que formatar um título em HTML requer abrir e fechar tags (ex: ⁠ <h1>Meu Título</h1> ⁠), o que é mais trabalhoso.  
   Em Markdown, basta adicionar um ⁠ # ⁠ na frente do texto, tornando o processo muito mais simples e direto.

 7.⁠ ⁠*Qual a extensão de arquivo comum para documentos Markdown, e onde o vídeo exemplifica sua utilização prática?*  
   A extensão de arquivo comum para documentos Markdown é ⁠ .md ⁠.  
   O vídeo exemplifica sua utilização prática nos arquivos ⁠ README.md ⁠ encontrados em repositórios do GitHub, como o da biblioteca Pytorch.

 8.⁠ ⁠*Por que as empresas e desenvolvedores utilizam Markdown para criar arquivos README.md em projetos de código aberto?*  
   Eles utilizam Markdown porque facilita a criação de documentação clara, formatada e de fácil leitura para explicar o projeto, como instalá-lo, como usá-lo, etc.  
   Isso torna a visualização e compreensão do projeto mais acessível para outros desenvolvedores.

 9.⁠ ⁠*O que o vídeo sugere para quem está construindo um portfólio de projetos no GitHub em relação ao uso do Markdown?*  
   O vídeo recomenda que, ao construir um portfólio de projetos no GitHub, o desenvolvedor crie um arquivo ⁠ README.md ⁠ usando Markdown para explicar o projeto de forma didática, formatada e visualmente atraente.

10.⁠ ⁠*Qual a principal vantagem do Markdown, segundo o vídeo, para a construção de documentos de texto em programação?*  
    A principal vantagem é a capacidade de construir textos de forma muito mais rápida e prática, aplicando formatações de maneira simples, o que é ideal para explicar conceitos de programação e documentar projetos.

## Glossário de Termos-Chave

•⁠  ⁠*Markdown:* Linguagem de marcação leve que permite formatar texto simples para produzir documentos com estilos visuais (títulos, negrito, itálico, listas) de forma rápida e intuitiva.  
•⁠  ⁠*Linguagem de Marcação:* Sistema para anotar um documento de forma que possa ser diferenciado do texto, com o objetivo de formatá-lo, estruturá-lo ou adicionar informações semânticas.  
•⁠  ⁠*HTML (HyperText Markup Language):* Linguagem padrão para criar páginas web, utilizada para estruturar o conteúdo de uma página (parágrafos, títulos, imagens) e definir seu layout.  
•⁠  ⁠*Front-end:* Parte de um website ou aplicação com a qual o usuário interage diretamente. Markdown é útil para formatar o conteúdo visual dessa parte.  
•⁠  ⁠*Jupyter Notebook:* Aplicativo web interativo que permite criar e compartilhar documentos que contêm código, equações, visualizações e texto explicativo (frequentemente formatado com Markdown).  
•⁠  ⁠*README.md:* Arquivo de texto comum encontrado em repositórios de projetos de software (especialmente no GitHub), escrito em Markdown, que fornece informações essenciais sobre o projeto, como descrição, instalação e uso.  
•⁠  ⁠*GitHub:* Plataforma de hospedagem de código-fonte e arquivos com controle de versão usando Git, amplamente utilizada para desenvolvimento de software colaborativo e projetos de código aberto.  
•⁠  ⁠*Repositório Online:* Local centralizado onde o código-fonte de um projeto e seus arquivos relacionados são armazenados e gerenciados, permitindo que vários desenvolvedores colaborem.  
•⁠  ⁠*Código Aberto (Open Source):* Software cujo código-fonte está disponível publicamente para que qualquer pessoa possa ver, modificar e distribuir.  
•⁠  ⁠*Formatação de Texto:* Processo de aplicar estilos visuais a um texto (como tamanho da fonte, cor, negrito, itálico, alinhamento) para melhorar sua legibilidade e aparência.  
•⁠  ⁠*Biblioteca (de Programação):* Conjunto de funções, classes e objetos pré-escritos que os programadores podem usar para desenvolver aplicativos e economizar tempo. O vídeo menciona a biblioteca Pytorch.  
•⁠  ⁠*Asterisco ( ou *):* Símbolo usado em Markdown para indicar texto em itálico (com um asterisco) ou *negrito* (com dois asteriscos).  
•⁠  ⁠*Hashtag (#):* Símbolo usado em Markdown para indicar títulos ou cabeçalhos, com o número de hashtags definindo o nível do título (ex: ⁠ # ⁠ para H1, ⁠ ## ⁠ para H2).
